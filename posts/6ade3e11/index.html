vscode的claude code插件使用deepseekV4模型

发表于2026-05-09|更新于2026-05-09
|总字数:934|阅读时长:3分钟|浏览量:

前言

Claude Code 插件默认走 Anthropic 的模型，但如果你想换成 DeepSeek V4 降低成本，这篇就是一份实操笔记，涵盖从配置到踩坑的全过程。

下载插件

首先，打开 VS Code 的插件市场，搜索 “Claude Code for VS Code”，找到并安装它。

配置插件

点击vscode左下角的设置小齿轮，选择Extensions，找到Claude Code，点击Settings json按钮，将以下内容粘贴进去。


下载CC Switch

CC Switch 是一个开源的桌面 GUI 工具，专门用来管理 Claude Code 的配置。它的核心作用是：

  1. 一键切换模型供应商 — 在 Claude / DeepSeek / GLM / Kimi / 通义千问等之间随意切，不用手写 JSON 配置文件
  2. 管理 API Key、代理地址、MCP、Skills — 图形界面操作，比手动改环境变量方便很多
  3. 本地请求转发 — 拦截 Claude Code 发出的 API 请求，转发到你指定的第三方模型接口

本质上它在本地跑了一个代理服务，默认地址是 http://127.0.0.1:11435，Claude Code 的请求先发到这里，CC Switch 再根据你选的渠道转发到对应的模型。

GitHub 地址：https://github.com/farion1231/cc-switch

下载完成后安装到合适的路径然后打开就行了。

总结

整个配置流程走下来其实不复杂：装好 Claude Code 插件和 CC Switch，填上 DeepSeek 的 API Key，切一下模型就能用了。Claude Code 强大的 Agent 能力（自动读写文件、执行终端命令、任务规划等）配合 DeepSeek V4 的推理能力和亲民价格，算是一个性价比很高的组合。

与其他 VS Code 插件接 DeepSeek 的区别

市面上还有其他 VS Code AI 插件也能接 DeepSeek，但它们和「Claude Code + DeepSeek」的定位不太一样：

Cline（原 Claude Dev）
开源、原生支持 DeepSeek，配置更简单，直接填 API Key 就行，不需要 CC Switch 这类中间工具。但它的 Agent 工具链（文件操作、终端执行、任务拆解）相比 Claude Code 还是弱一些，处理复杂工程任务时容易跑偏。

Continue
更像是一个代码补全和对话工具，侧重点是代码理解和内联补全，适合边写边问的场景。它没有 Claude Code 那种「全自动执行任务」的 Agent 能力，更像一个智能助手而不是一个能独立干活的开发者。

GitHub Copilot / Copilot Chat
只能用微软自己的模型，没法切换到 DeepSeek，直接不用比。

通义灵码 / 文心快码 等国产插件
部分支持配置第三方模型，但主要还是为自家模型优化。Agent 能力和 Claude Code 差距明显，更适合作为代码补全和简单问答工具。

总结一句话：如果你只是想要代码补全和聊天，Cline 或 Continue 接 DeepSeek 够用了；但如果你想要 Claude Code 级别的全自动 Agent 体验又不想付 Anthropic 的价格，那 Claude Code + CC Switch + DeepSeek V4 是目前最划算的选择。
